今回は、Astro を使ってブログを作ってみようと思い、導入手順を簡単にまとめました。  
本記事では、開発環境の準備からプロジェクト作成、GitHub Pages への公開方法の確認までをメモとして整理しています。

## はじめに

Astro でブログを作成してみたくなったため、まずは最低限のセットアップを行いました。  
本記事は、その際に実施した内容を簡単にまとめたものです。

まだ試しに触ってみる段階なので、細かい構成やデザインについては扱っていません。

## 環境作成について

当初は、試験的な利用ということで devcontainer を使って環境を構築しようと考えていました。

しかし実際に確認してみると、Astro はプロジェクト単位でインストールされるフレームワークであり、特別な実行環境を用意しなくても問題ありませんでした。

そのため、今回は devcontainer は使用せず、通常のローカル環境でプロジェクトを作成しています。

## プロジェクト作成

Astro のプロジェクト作成は非常に簡単です。

以下のコマンドを実行するだけで、セットアップが開始されます。

```bash
npm create astro@latest
```

コマンドを実行すると、対話形式でいくつか質問されるので、表示される選択肢に従って進めていくだけで初期プロジェクトが作成されます。

特に難しい設定はなく、テンプレートや TypeScript の利用有無などを選択する程度で完了します。

## GitHub Pages でサイトを公開する

作成した Astro のサイトは、GitHub Pages で公開することができます。

公式ドキュメントに手順がまとまっているため、以下のページを参照しながら設定を行います。
[Deploy your Astro Site to GitHub Pages | Docs](https://docs.astro.build/en/guides/deploy/github/)

今回は、次の点を確認しました。
	•	Astro のサイトを GitHub Pages にデプロイできること
	•	公式ドキュメントに手順が用意されていること

実際のデプロイ作業は、上記ドキュメントを参照しながら進める予定です。
